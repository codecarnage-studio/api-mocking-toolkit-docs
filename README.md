# API Mocking Toolkit Docs

This repository contains the documentation site for the **API Mocking Toolkit** Unity asset.

- Live docs: https://api-mocking-toolkit.codecarnage.com/

The site is built with [Docusaurus 3](https://docusaurus.io/).

---

## What is API Mocking Toolkit?

API Mocking Toolkit lets you:

- Mock HTTP/REST backends directly inside Unity
- Capture JSON responses and replay complete sessions
- Simulate errors, latency and edge cases
- Work completely offline while the backend is still in development

You can use it to:

- Unblock gameplay and UI work before backend APIs exist
- Test pagination, variations and multi-step flows via **Response Strategies**
- Switch cleanly between Dev/Staging/Prod via **Environments & Variables**
- Record and replay complex play sessions for debugging
- Import/export OpenAPI specs to collaborate with backend teams

For the full feature overview, see the docs:

- https://api-mocking-toolkit.codecarnage.com/docs/core-features

## Reporting issues & support

Use **GitHub issues on this repo** for:

- **Bugs in the API Mocking Toolkit Unity asset**
- **Feature requests** for the asset or its tooling
- **Documentation problems** (missing info, unclear sections, typos)

When reporting a Unity/runtime issue, it really helps if you include:

- Unity version (e.g. 2021.3.35f1)
- API Mocking Toolkit version
- Platform (Editor only, Windows, macOS, etc.)
- Repro steps (what you did, what you expected, what happened)
- Screenshots or small video/gif if relevant

For doc issues, please link to the page/section and describe what’s wrong or what you expected to find.

If you need private support, you can also reach us at **support@codecarnage.com**.
